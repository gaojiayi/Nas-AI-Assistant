# 飞牛OS ONVIF协议监控对接完整指南

飞牛 OS（fnOS）作为一款新兴的国产 NAS 操作系统，以其轻量和底层基于 Debian 的特性受到很多玩家的喜爱。

## 背景说明

目前（截至最新的版本状态），飞牛 NAS 系统本身并没有像群晖（Surveillance Station）或威联通那样内置官方的监控摄像头管理套件。

但是，因为飞牛 NAS 拥有非常完善的 Docker（容器）支持，所以对接 ONVIF 协议摄像头的标准、也是最强大的流程是：**在飞牛 NAS 中通过 Docker 部署第三方的 NVR（网络视频录像机）系统，然后通过该系统对接摄像头**。

以下是完整的实操流程指南：

## 第一步：准备摄像头端（非常关键）

不同品牌的摄像头（如海康、大华、TP-LINK等），ONVIF 协议默认可能是关闭的，需要先开启。

### 配置清单：

- **固定 IP 地址**：进入摄像头的后台管理页面，将摄像头的 IP 地址设置为静态 IP，避免路由器重启后 IP 变化导致断连。

- **开启 ONVIF 协议**：
  - **海康威视**：进入 `配置 -> 网络 -> 高级配置 -> 集成协议`。勾选"启用 ONVIF"，然后必须点击"添加"按钮，创建一个专门用于 ONVIF 的用户名和密码（建议权限设为管理员）。
  - **大华/TP-LINK**：同样在网络设置或安全设置中找到 ONVIF/API 选项并启用，确保有对应的账号密码。

- **确认端口**：记下 ONVIF 的端口号（海康通常是 80，大华可能是 80，部分品牌是 8899 或 2020，具体看后台显示）。

- **视频编码**：建议先将摄像头的视频编码格式改为 H.264。虽然 H.265 更省空间，但很多开源 NVR 在网页端直接播放 H.265 会报错。

## 第二步：在飞牛 NAS 准备存储空间

你需要为监控录像专门分配一个存储位置。

### 操作步骤：

1. 打开飞牛 NAS 的文件管理器
2. 在你希望存放录像的存储空间下，新建一个共享文件夹，例如命名为 `Surveillance` 或 `NVR_Data`

## 第三步：在飞牛 NAS 中安装 NVR 系统（通过 Docker）

这里推荐两款最主流的开源监控软件，你可以二选一：

- **Agent DVR (iSpy)**：图形界面友好，设置简单，适合新手
- **Frigate**：配置稍微复杂（需要写 YAML 文件），但支持接入 Google Coral 或显卡进行极其强大的 AI 人像/物体识别，适合极客玩家

### 👉 以最易上手的【Agent DVR】为例的操作流程：

1. **打开飞牛 NAS 的 Docker（容器）管理界面**

2. **下载镜像**：在镜像仓库中搜索 `doitandbedone/ispyagentdvr` 并拉取（Pull）最新版（latest）

3. **创建容器**：
   - **容器名称**：起个名字，比如 `AgentDVR`
   - **网络设置**：建议选择 `Host（主机模式）`，这样最容易搜到局域网内的摄像头；或者选择 `Bridge`，但需要映射端口（默认Web端口是 8090）
   - **存储空间（路径映射）**：这一步极其重要，否则重启后录像全丢！
     - 将你刚才在飞牛里建的文件夹 `/volX/XXX/Surveillance` 映射到容器内部的 `/agent/Media/XML`（存放配置文件）
     - 映射到容器内部的 `/agent/Media/WebServerRoot/Media`（存放录像视频）

4. **启动容器**

## 第四步：对接 ONVIF 摄像头

1. **访问后台**：在浏览器输入飞牛 NAS 的 IP 地址加端口号，例如：`http://飞牛NAS的IP:8090`，进入 Agent DVR 的控制台

2. **添加摄像头**：
   - 点击界面上的"服务器"图标 -> 选择 "添加设备" (Add Device)
   - 选择 "视频源" (Video Source)
   - 在弹出的窗口中，视频源类型选择 "ONVIF"

3. **输入配置信息**：
   - **用户名/密码**：输入你在第一步里创建的摄像头的 ONVIF 账号和密码
   - **URL/IP**：输入摄像头的 IP 地址（例如 `http://192.168.1.100`）
   - 点击 "获取视频流" (Get Video URLs) 按钮

4. **选择通道**：系统会自动连接摄像头并列出可用的 RTSP 视频流（通常分为主码流 Main Stream 和子码流 Sub Stream。主码流高清用来录像，子码流低清用来在九宫格里预览）。选中你要的视频流，点击确定

5. **完成**：此时你应该能在飞牛 NAS 上的 Agent DVR 网页里看到摄像头的实时画面了

## 进阶建议

### 硬件解码
如果你用的是 Intel 处理器的飞牛 NAS，想要降低录像时的 CPU 占用，可以在 Docker 创建容器时，将 NAS 的核显设备（通常路径是 `/dev/dri`）映射进容器，并在 NVR 软件中开启硬件解码（VAAPI 或 QuickSync）。

### 时间同步
确保飞牛 NAS 的系统时间与摄像头内部的系统时间是一致的（最好都配置同一个 NTP 授时服务器），否则 ONVIF 握手验证可能会因为时间差过大而失败。
